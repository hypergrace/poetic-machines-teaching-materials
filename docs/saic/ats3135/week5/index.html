Unit 1 Project

After learning how to integrate live data streams via APIs into our sketches, your first project will be display visually (or otherwise) the remaining carbon budgeted before a global increase in temperature of 1.5 or 2 degrees Celsius.

requirements

  • use p5*js techniques you have learned from the teach-in. expand your toolkit as necessary
  • use the co2 clock API provided in the tutorial below
  • you may use text, but you must also use graphical drawing to visualize some data taken from the API.

considerations

  • The number of remaining co2 budget is really big. There aren't enough pixels on the screen to represent it. A 4K display has 8,294,400 pixels, compared to the number of tons of co2 left when writing this: 288,504,002,234. We would need ~35 thousand 4k monitors just to display each ton of carbon as a single pixel. How will you make a meaningful reduction?
  • The data changes overtime, so you must think it terms of animation rather than a still image.
  • data visualization always makes some aspects of a phenomenon more palpable and rendering other aspects more opaque. What does your visualization focus on? What are the limitations of your visualization?
  • If your visualization is indistinguishable from a counter moving downward, is there some context to add?
  • make a decision about whether you want a "naïve viewer" to "get" what's being displayed.
  • consider the "carbon liberation front" front from McKenzie Wark's essay. Without copying her technique, how can you leverage the approach in your visualization?

Tutorial

This tutorial provides some usage instructions for accessing carbon data in p5.js. This library is developed by Olaf Val and pulls from the Mercator REsearch Institute on Global Commons and Climate Change. You can see their visualization of the climate clock here.

The GitHub page has an extensive reference you should continue to look at as you prototype your visualization. Here are a few key upshots to keep in mind.

View the code embedded below. Inspect the "sketch files" using the collapsible window on the left. Notice that co2budget.js has been added. This contains all the code you need to call co2 functions.

Inspect the index.html file. Notice on line 6 and 7 that a reference to the co2budget.js file has been added. You need to take this step in order to call the co2 functions!

Inspect the code below. Notice:

  • I use two arrays.
    • times keeps track of the current clock's years, months, etc.
    • I also use a two dimensional array called circles. This holds all the TimeCircle JS objects.
  • I used the Class keyword to make a template for each JS Object I want to make. Each TimeCircle object has a unique X and Y position and a radius that corresponds to its time unit (Y, M, D, etc.). As a result, the randomly generated values each TimeCircle object persists between frames, so the circle can stay in the same place.
  • I made my own function called updateClock(). This encapsulates all the clock updating operations: clearing the times array, calling each function, and adding them into the array
  • notice my reliance on for loops. For dealing with large numbers of operations, there's no way around them.
